## Context

The FretboardViewer (`Fretboard2.vue`) manages a fretboard visualization with three control panels. The middle panel contains a "Subset enabled notes" section powered by `NoteSelectionList.vue`, which renders a flat table of checkboxes — one per scale degree. Users toggle notes on/off to explore subsets within a tuning. The selection state lives as a boolean array (`subsetEnabled`) on `Fretboard2`.

Currently there is no EDO detection — the component works purely with ratio arrays. EDO information exists only in the template name string (e.g., `"31edo - Guitar - Standard Tuning"`) and in the raw note text format (e.g., `"5\31"`). Two 31edo templates are defined in `note-templates.js` with all 31 notes and Major/Minor scale presets.

To support this feature, three new properties will be added to note template objects in `note-templates.js`:
- `isEdo: true` — declares this template as an EDO-based tuning
- `edoIdx_Fifth: 18` — the chromatic index of the fifth (18 steps for 31edo, the meantone fifth ≈ 696.77¢)
- `useCircleOfFifthViewer: true` — opt-in flag enabling the circle-of-fifths component for this template

In the 31edo circle of fifths, the generator is `edoIdx_Fifth` (18) steps of 31. Starting from chromatic index 0 and repeatedly adding 18 (mod 31) produces the full 31-note cycle.

## Goals / Non-Goals

**Goals:**
- Display a circle-of-fifths visual selector when 31edo notes are active
- Keep the circle selector and the existing flat checkbox list in bidirectional sync
- Render notes in a proper circular layout with radially positioned checkboxes
- Detect 31edo note sets reliably

**Non-Goals:**
- Generalizing to arbitrary EDOs or circle-of-generators — only 31edo is supported in this change
- Replacing the existing flat list — the circle is an additional optional view
- Adding audio playback from the circle — notes are only toggled, not sounded
- Mobile-optimized responsive layout for the circle

## Decisions

### 1. New standalone component: `CircleOfFifths.vue`

**Decision**: Create a new `CircleOfFifths.vue` component in `src/FretboardViewer/`.

**Rationale**: The circle visualization is self-contained UI with its own layout logic. Embedding it in `NoteSelectionList.vue` would bloat that component with unrelated CSS and layout math. A standalone component keeps concerns separated and is easier to test and reuse.

**Alternatives considered**:
- Extend `NoteSelectionList.vue` with a circular mode — rejected because it would mix two distinct layout paradigms in one component.

### 2. EDO detection via template metadata properties

**Decision**: Add three new properties to note template objects in `note-templates.js`:
- `isEdo: true` — marks the template as EDO-based
- `edoIdx_Fifth: 18` — the chromatic step index representing the fifth interval (18 for 31edo)
- `useCircleOfFifthViewer: true` — opt-in flag to enable the circle-of-fifths component

The `CustomNotes` component will forward these properties through its `@change` event alongside the existing fields. `Fretboard2.vue` will check `useCircleOfFifthViewer` to decide whether to show the circle toggle. The `edoIdx_Fifth` value is passed to `CircleOfFifths.vue` to compute the note ordering.

**Rationale**: Explicit template metadata is the clearest and most extensible approach. It keeps detection logic out of the rendering layer, makes the intent self-documenting in the template definitions, and allows other EDOs to opt in to the circle viewer in the future by simply adding these properties.

**Alternatives considered**:
- Parse `noteTexts` entries for the `N\31` pattern — fragile, mixes detection logic with display data, and requires computing the fifth index separately.
- Match `selectedTemplate` string against known names — would break for custom-entered notes and requires maintaining a separate allowlist.

### 3. Circle of fifths ordering derived from `edoIdx_Fifth`

**Decision**: Compute the circle-of-fifths order at component initialization using the `edoIdx_Fifth` value from the template. The sequence is generated by `(i * edoIdx_Fifth) % noteCount` for `i` in `0..noteCount-1`. For 31edo with `edoIdx_Fifth = 18`, this produces:

Chromatic indices around the circle: 0, 18, 5, 23, 10, 28, 15, 2, 20, 7, 25, 12, 30, 17, 4, 22, 9, 27, 14, 1, 19, 6, 24, 11, 29, 16, 3, 21, 8, 26, 13.

**Rationale**: Computing from `edoIdx_Fifth` rather than hardcoding a constant array means the same component can work for any EDO that opts in via `useCircleOfFifthViewer` in the future — just pass a different `edoIdx_Fifth` value.

**Alternatives considered**:
- Hardcode a static constant array for 31edo only — simpler but not reusable if other EDOs are added later.

### 4. Circular layout via CSS absolute positioning + transforms

**Decision**: Position each note label + checkbox on a circle using absolute positioning with `transform: rotate(Xdeg) translateY(-Rpx)` for each of the 31 positions. The container is a square `div` with `position: relative` and a visible circle border via `border-radius: 50%`.

Each note element is placed at angle `(i / 31) * 360°` around the center, with the label rotated back to stay upright (`transform: rotate(-Xdeg)`).

**Rationale**: Pure CSS solution — no SVG, canvas, or external libraries. Works with Vue 2.6 and standard DOM. The approach is well-established for circular menus.

**Alternatives considered**:
- SVG-based layout — more flexible for drawing arcs and connecting lines but heavier to build and style; checkboxes would need to be foreign objects.
- CSS Grid with `clip-path` — insufficient control over precise radial positioning.

### 5. Bidirectional sync via props and events

**Decision**: `CircleOfFifths.vue` receives `selectedNotes` (boolean array, indexed by chromatic order), `noteTexts`/`noteNames`, and `edoIdx_Fifth` as props. When a user clicks a checkbox on the circle, it emits a `toggle(chromaticIndex)` event. `Fretboard2.vue` handles this event by updating `subsetEnabled` (the same array `NoteSelectionList` uses), which flows back down as props.

**Rationale**: This follows the existing Vue 2 pattern used by `NoteSelectionList` — parent owns the state, children emit events. No new state management needed. Both components react to the same `subsetEnabled` array.

**Alternatives considered**:
- Have CircleOfFifths directly mutate a shared array — violates Vue's one-way data flow.
- Event bus — overkill for sibling sync when the parent already owns the state.

### 6. Placement in the middle panel with a toggle

**Decision**: Add a checkbox "Show Circle of Fifths" below the `NoteSelectionList` in the middle `control-panel` of `Fretboard2.vue`. This checkbox is only visible when the active template has `useCircleOfFifthViewer: true`. When checked, the `CircleOfFifths` component renders below it, receiving `edoIdx_Fifth` from the template metadata.

**Rationale**: Keeps the feature discoverable without cluttering the UI for non-eligible tunings. A simple show/hide toggle is consistent with the existing "Show scale options" pattern. Using the explicit `useCircleOfFifthViewer` flag rather than inferring from `isEdo` allows fine-grained control over which templates show the circle.

### 7. Selection state refactoring in `Fretboard2.vue`

**Decision**: Lift `subsetEnabled` management slightly so that `Fretboard2` explicitly passes the current `subsetEnabled` array as a prop to both `NoteSelectionList` and `CircleOfFifths`, and both emit change events back to `Fretboard2`. Currently `NoteSelectionList` manages its own `selectedNotes` internally and only emits on change; this will be adjusted so that `NoteSelectionList` also accepts an `externalSelectedNotes` prop to stay in sync when the circle modifies the selection.

**Rationale**: True bidirectional sync requires both components to reflect the same state. Without this, toggling a note on the circle would not immediately update the flat list's checkboxes.

## Risks / Trade-offs

- **31 items on a circle is visually dense** → Mitigate with small font, tight spacing, and adequate circle radius (~300px diameter). Test readability at that density.
- **Long note names with accidentals (‡, d, b, #) vary in width** → Use fixed-width containers for each note position to prevent layout jitter.
- **`NoteSelectionList` internal state sync** → Introducing an external prop for selected notes adds coupling. Mitigate by making the prop optional — component falls back to self-managed state when no external override is provided.
- **Only supporting 31edo limits reusability** → Acceptable trade-off for this change. The component architecture (constant array of circle positions, generic checkbox sync) could be extended to other EDOs later by parameterizing the generator and count.
